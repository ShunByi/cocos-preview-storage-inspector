# Cocos 预览本地存储

## 作者说明

该插件是我用 **AI 辅助生成**的，初衷很简单：在 **Cocos 网页预览**时，能直接 **改、存到本地的数据**，省得本来只想动 **存档里某一块或改个值**，还要 **专门去改代码**；只想 **清掉这个项目落在浏览器里的数据** 时，也 **不想靠删浏览记录** 那种一锅端的方式。

后续 **好不好用、实不实用与我无瓜**，你自己掂量着用，风险自负。

下面整篇 README（除本段作者说明外）也是 **AI 写的**，**自己参考着怎么用** 就行。

---

### 扩展是干什么的（一句话）

面向 **Chrome** 里预览 **Cocos Creator（2.4 / 3.x）** 等页面时，在开发者工具里集中查看、搜索并编辑 **localStorage**、**sessionStorage**、**IndexedDB**（含多 iframe）。**仅建议在本地开发使用**，勿用于敏感生产数据。

当前版本见根目录 `manifest.json` 里的 `version`。

---

## 安装 / 导入 Chrome

1. 克隆或下载本仓库到本地，保证文件夹内包含 `manifest.json`。
2. 打开 Chrome，地址栏输入 `chrome://extensions` 并回车。
3. 打开右上角 **「开发者模式」**。
4. 点击 **「加载已解压的扩展程序」**。
5. 选择本仓库根目录（选中包含 `manifest.json` 的那一层文件夹）。
6. 加载成功后，扩展会出现在列表中；之后若你改了代码，可点击该扩展卡片上的 **刷新** 图标重新加载。

---

## 打开面板

1. 用 Chrome 打开你的 Cocos 预览页（或任意需要调试的页面）。
2. 按 **F12** 或 **Ctrl+Shift+I**（macOS：**Cmd+Option+I**）打开 **开发者工具**。
3. 在 DevTools 顶部标签栏中找到 **「Cocos存储」** 面板并点击打开。

若看不到该标签，确认扩展已启用，并尝试关闭再打开 DevTools。

## 具备哪些功能

| 能力 | 说明 |
|------|------|
| **多 iframe** | 工具栏下拉框可选择不同 **frame**，分别读取各框架下的存储（适配含 iframe 的预览页）。 |
| **localStorage / sessionStorage** | 分标签列出键值；支持将值按 **JSON 树** 展示（对象/数组）。 |
| **IndexedDB** | 列出数据库、对象仓库及抽样条目（键与可序列化值的预览；大库有数量上限，仅作调试参考）。 |
| **查找** | **存储键名**：按键模糊过滤；**JSON 内字段名**：在嵌套 JSON 中按字段名匹配。 |
| **查看 / 修改** | 列表顶栏可切换全局「查看 / 修改」；大窗详情内另有独立切换，仅作用于该弹窗。 |
| **JSON 树操作** | 展开/收起节点、**全展/全收**、**详情**（大窗口编辑整条或子树）、子树内 **返回** 上一级详情、**复制全部**、**重置**、增删改键与数组项、清空数组内容等。 |
| **刷新** | 从页面重新拉取各 frame 的存储快照。 |

## 使用方法简述

### 工具栏

- **刷新**：重新采集当前选中 frame 的存储数据。
- **页面 / frame**：切换要查看的 iframe（与主文档同级子框架）。
- **查找**：选择「存储键名」或「JSON 内字段名」，在输入框中输入关键词过滤列表。

### 标签页

- **localStorage**：持久化键值。
- **sessionStorage**：会话键值。
- **IndexedDB**：数据库与对象仓库的只读/预览型列表（具体能力与页面安全上下文有关）。

### 列表与 JSON 行

- 对解析为 JSON 对象或数组的条目，会显示树形结构。
- **详情**：打开大窗口，查看或编辑整条存储或某一子对象/子数组；在子路径上可继续点 **详情** 深入，使用 **返回** 回到同一键下的上一层视图。
- **修改** 类操作在 **「修改」** 模式下可用；仅查看时相应按钮会受限（以界面提示为准）。
- 标量类型（字符串、数字、布尔等）可通过行内 **修改** 等入口在弹窗中编辑。

### 弹窗与快捷键习惯

- **确定 / 取消**：按弹窗底部按钮；部分多层编辑会先回到上一层而非直接关闭整个 DevTools 弹窗。
- 点击遮罩空白处行为与扩展内 **取消/返回** 逻辑一致（以当前实现为准）。

---

## 权限说明

- **`scripting`**：在已调试标签页的各 frame 中执行脚本，用于读取 `localStorage` / `sessionStorage` / `IndexedDB` 元数据与内容。
- **`host_permissions: <all_urls>`**：对上述页面注入与采集；扩展为 **DevTools** 类型，仅在开发者主动打开 DevTools 并使用本面板时配合使用。

---

## 注意事项

- 本工具会直接读写页面存储，误操作可能导致存档或配置丢失，**请先备份**或使用版本管理。
- IndexedDB 展示为抽样与序列化预览，**不等同于**完整数据库导出。
- 若页面跨域隔离或存储不可用，部分数据可能无法显示，属浏览器安全策略限制。
